# Module 04 — Zo Builds Itself

> **Phase: MIRROR** | Time: ~30 min | You'll need: Claude (claude.ai) + Zo (zo.computer)

---

## The Big Idea

You don't code an agent. You **describe** it. Claude designs the blueprint. Zo builds the system. You iterate.

```
  YOUR CONTEXT           CLAUDE                  ZO
  ┌─────────┐          ┌─────────┐           ┌─────────┐
  │ Who you  │          │ Designs │           │ Creates  │
  │ are,     │   --->   │ the     │   --->    │ all 6    │
  │ what you │          │ .zo     │           │ pieces   │
  │ need     │          │ config  │           │ from the │
  │          │          │ file    │           │ config   │
  └─────────┘          └─────────┘           └─────────┘
```

This module walks you through the full workflow: dump your context into Claude, get a structured `.zo` config file, paste it into Zo, and watch the system come online.

## Step 1: Gather Your Context

Before you talk to Claude, collect the raw material. You don't need to organize it — just dump it. Claude will find the patterns.

Gather any combination of:
- Your daily routine (what do you do every morning, every week?)
- Your current projects/clients
- Emails you've sent that show your voice
- Templates you use regularly
- Tools you use (Gmail, Slack, Notion, etc.)
- Things you wish someone would just handle for you

**The messier the better.** Claude is good at finding structure in chaos.

## Step 2: The Claude Blueprint Prompt

Open [claude.ai](https://claude.ai). Paste your raw context, then follow it with this prompt:

---

### Prompt: Generate a Zo Agent Blueprint

```
I'm building a personal AI agent on Zo Computer. I've given you my raw context
above — my work, my tools, my routines, my communication style.

Based on everything I've shared, generate a complete .zo configuration file
that I can paste into Zo to bootstrap my agent. Use this exact structure:

---

# ZO AGENT CONFIGURATION
# Generated by Claude from {user_context}

## BIO
Write a 2048-character-max bio that includes:
- Who I am and what I do
- My active projects/clients (by name)
- Communication preferences (which channel for what)
- Current priorities
- Timezone
- Voice/tone description

## RULES

### Always-On Rules
List 5-7 rules the agent must always follow. Focus on:
- Forbidden actions (what it must never do without permission)
- Voice/tone constraints
- Communication cadence boundaries
- Decision shortcuts (e.g., "lgtm" = approval)

### Conditional Rules
List 3-5 rules triggered by specific conditions.
Format: When [condition] -> [instruction]

## PERSONA

### Primary Persona
- Name: [descriptive name]
- Prompt: [full persona prompt including mission, voice, operating mode]
- Recommended channels: [which channels this persona should be active on]

## SKILLS

### Skill 1: [name]
- Description: [what it does + trigger phrases]
- Steps: [numbered procedure]
- Tools needed: [which Zo tools]
- Definition of done: [how the agent knows it's finished]

### Skill 2: [name]
(repeat for each identified workflow — aim for 3-5 skills)

## AUTOMATIONS

### Automation 1: [name]
- Schedule: [RRULE format, e.g., FREQ=DAILY;BYHOUR=7;BYMINUTE=30]
- Delivery: [sms / email / telegram / slack]
- Instruction: [complete instruction with all context, file paths, tools]

(repeat for each — aim for 2-4 automations)

## FILE STRUCTURE

Suggest a filesystem layout for /home/workspace/ that supports the
skills and automations above. Include:
- Client/project folders
- Templates folder
- A seed document for each file the agent will reference

---

Be specific. Use real names, real tools, real schedules from my context.
Don't ask me questions — make your best inference from what I've shared.
I'll iterate after I see the first draft.
```

---

## Step 3: Review the Blueprint

Claude will generate a complete `.zo` config. Read through it and check:

- [ ] Does the Bio sound like you?
- [ ] Are the Rules things you actually want enforced?
- [ ] Does the Persona match how you want the agent to behave?
- [ ] Do the Skills capture your real workflows?
- [ ] Are the Automations on schedules that make sense?

**Iterate with Claude.** Say things like:
- "The voice is too formal — make it more direct and warm."
- "Add a rule: never text me on Sundays unless it's urgent."
- "The morning briefing should also check my Linear board."
- "Remove the weekly report — I don't need that yet."

## Step 4: The Zo Bootstrap Prompt

Once your `.zo` config is ready, go to [zo.computer](https://zo.computer) and paste this into chat:

---

### Prompt: Bootstrap from .zo Config

```
I have a complete agent configuration below. Please set up my Zo
by creating each piece:

1. Update my Bio with the BIO section
2. Create each Rule from the RULES section
3. Create the Persona from the PERSONA section
4. Create each Skill as a SKILL.md file in /home/workspace/Skills/
5. Create each Automation from the AUTOMATIONS section
6. Create the file/folder structure from the FILE STRUCTURE section

After creating everything, give me a summary of what you built:
- How many rules created
- Which persona is active
- Which skills are installed
- Which automations are scheduled (with their next run time)
- What folders were created

Here's the configuration:

[PASTE YOUR .ZO CONFIG HERE]
```

---

## Step 5: Verify

After Zo finishes, check each piece:

```
Show me my current Bio.
List all my rules.
List my personas.
List my skills.
List my automations.
Show me the file tree at /home/workspace/
```

If anything is off, just tell Zo: "Update rule #3 to say..." or "Change the automation schedule to 8am instead of 7:30am."

## How the Self-Build Loop Works

What you just did is the pattern you'll use forever:

```
  1. CONTEXT DUMP       You throw your messy reality at Claude
                         |
  2. CLAUDE DESIGNS     Claude finds the structure, generates the .zo config
                         |
  3. ZO BUILDS          Zo creates all six primitives from the config
                         |
  4. YOU USE IT         Use the agent for real work. Notice what's off.
                         |
  5. CLAUDE DIAGNOSES   Tell Claude what happened: "It texted me at 6am"
                         |                         "The briefing missed my calendar"
  6. ZO FIXES           Tell Zo the fix: "Change the schedule to 8am"
                         |                "Add Google Calendar to the briefing skill"
                         |
                       REPEAT from step 4
```

The `.zo` config file is your **source of truth**. Update it as you iterate. In Module 06, you'll learn how to export it to GitHub so it's version-controlled and portable.

## Follow-Up Prompts for Claude

When iterating on your agent, use these with Claude:

| Prompt | When to Use |
|---|---|
| "What patterns do you see in how I do this?" | When you want Claude to find structure in your work |
| "What would a system need to know about me to do this well?" | When the agent's output is too generic |
| "What should it handle on its own vs. ask me about?" | When you're defining the approval boundary |
| "Write the instructions as if briefing a new employee." | When a skill's steps are too vague |
| "What could go wrong with this system? What's missing?" | When you want a stress test of your config |

## Troubleshooting

| Problem | Fix |
|---|---|
| Generic output from the agent | Bio needs more detail about you |
| Wrong steps in a skill | Skill instructions need updating — be more specific |
| Didn't check email/calendar | Integration not connected, or not mentioned in the skill |
| Did something you didn't want | Add a Rule to prevent it |
| Automation didn't run | Check the schedule (RRULE format) and delivery channel |
| Persona voice is off | Rewrite the persona prompt with examples of your actual voice |
